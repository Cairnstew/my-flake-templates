# Minecraft Forge Mod — Nix Flake Template

A fully dynamic, reproducible dev environment for Minecraft Forge mods.
All project settings flow from **one file**: `mod-config.nix`.

---

## Quick Start

```bash
# 1. Use as a flake template (recommended)
nix flake new -t github:yourname/forge-mod-template my-mod
cd my-mod

# 2. Edit your mod settings
$EDITOR mod-config.nix

# 3. Enter the dev shell
nix develop

# 4. Build the mod jar
nix run .#build

# 5. Launch a test client
nix run .#client
```

---

## Configuration (`mod-config.nix`)

| Field | Description |
|---|---|
| `modId` | Unique snake_case identifier |
| `modName` | Human-readable display name |
| `modVersion` | Semantic version string |
| `modGroupId` | Java group ID (reverse domain) |
| `authors` | List of author strings |
| `modDescription` | Short description |
| `homepage` | Project URL |
| `license` | `pkgs.lib.licenses.*` value |
| `gradleMemory` | JVM heap for Gradle (e.g. `"2G"`) |
| `optionalDependencies` | List of optional mod IDs |
| `requiredDependencies` | List of required mod IDs |
| `extraPackages` | Function `pkgs → [pkg]` for shell extras |
| `shellHookExtra` | Bash string appended to shell hook |

### Switching Minecraft / Forge Versions

`mod-config.nix` ships with five presets. To switch, change one line:

```nix
# In mod-config.nix
preset = presets.mc1_20_4;   # was mc1_20_1
```

Available presets: `mc1_20_4`, `mc1_20_1`, `mc1_19_2`, `mc1_18_2`, `mc1_16_5`.

Java is selected **automatically** based on the Minecraft version:

| Minecraft | Java |
|---|---|
| ≤ 1.16 | JDK 8 |
| 1.17 | JDK 16 |
| 1.18 – 1.20.4 | JDK 17 |
| ≥ 1.20.5 | JDK 21 |

## Nix Outputs

| Command | Effect |
|---|---|
| `nix develop` | Enter the dev shell |
| `nix run .#build` | Build the mod jar |
| `nix run .#client` | Launch Minecraft client |
| `nix run .#server` | Launch a dedicated server |
| `nix run .#genData` | Run data generators |
| `nix build` | Produce the jar in `result/lib/` |
| `nix flake check` | Validate the flake |

```
├── settings.gradle
├── src/
│   └── main/
│       ├── java/com/example/examplemod/
│       │   └── ExampleMod.java
│       └── resources/
│           ├── META-INF/mods.toml   # properties injected by Gradle
│           └── pack.mcmeta
└── .github/workflows/build.yml # CI via Nix
```

## Adding Mod Dependencies

1. Add the Maven repo to `build.gradle` under `repositories {}`.
2. Add the `compileOnly`/`runtimeOnly` line to `dependencies {}`.
3. For JEI/Curios style soft deps, add a `[[dependencies.modid]]` block in `mods.toml`.
4. List the mod ID in `optionalDependencies` or `requiredDependencies` in `mod-config.nix` for documentation/env-var purposes.

---

## Reproducibility

Because `gradle.properties` is **generated by Nix** on every `nix develop` or `nix run`, your Forge version, MC version, mod metadata, and Java version are all locked to the values in `mod-config.nix`. Committing `mod-config.nix` and `flake.lock` gives every contributor an identical environment.
